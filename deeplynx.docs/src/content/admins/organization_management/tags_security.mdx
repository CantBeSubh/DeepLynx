# Tags and Security Labels

Nexus also supports **attribute based access controls (ABAC)** where resources within parent organizations
are assigned tags and security labels that are passed down the heirarchy to projects.
Currently, organization admins have the ability to create, edit, and delete tags.

**Note:** Security labels will be enabled in a future release.

## Understanding Standard Tags and Security Labels

**Tags** can be added to organizations to classify their projects and resources.
In a future release, **security labels** will be enabled for organizations to enforce custom rules.
Security labels will also be passed down the heirarchy to projects.

## Creating Custom Tags

1. Navigate to the Organization Management page by clicking the slider bar icon on the outtermost toolbar.
2. On the Tags and Security Labels tab of the Organization Management page, click the "New Tag" button.

<Image
  src="/docs/images/create-tag-org.png"
  alt="Creating Tags UI"
  width={800}
  height={600}
/>

3. Enter the new tag's name and click "Create Tag".
4. The new tag will appear in the organization tags widget of the Tag Management page!

## Editing Tags

Organization admins can edit the names of existing organization tags.

1. Click the "Edit" button next to the selected tag inside the Organization Tags widget.

  alt="Pre Editing Tags UI"
  width={800}
  height={600}
/>

2. Update the tag's information in the Edit Tag modal and hit save.

## Locking Tags

The organization admin has the ability to lock organization tags by toggling the lock icon on the Organization Tags Widget.
This prevents users from updating tags on a project level.

## Removing Tags

The organization admin has the ability to remove organization tags by clicking the "Delete" button of Organization Tags Widget.

**Note:** Once a tag is archived it can no longer be used. It must be restored to be reapplied to an organization.
